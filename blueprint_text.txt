Data Flow
The canonical data flow for a complete user session proceeds as follows. The user opens the Flutter app
and authenticates via Firebase Auth (JWT token issued, stored in SecureStorage). The user speaks in
Telugu: audio is captured by the Flutter record plugin, compressed to AAC, and sent as binary WebSocket
frames to the FastAPI backend. The ASR agent (Whisper Large v3 via Hugging Face Inference API)
transcribes the audio to text with automatic language detection. The transcript is passed to the LangGraph
Master Agent, which classifies intent using Llama 3.3 70B via Groq (design_request).
The Master Agent routes to the parameter extraction node, which parses body type, occasion (wedding),
budget (under 5000 rupees), color preference (red/gold), and cultural context (Hyderabad, Telugu). The
agent checks the user profile in PostgreSQL: no body measurements exist, so it requests photos. The
Flutter app launches the guided camera capture with silhouette overlay, blur detection, and brightness
check. Two photos (front and side) are compressed to 800px JPEG and uploaded to Supabase Storage.
The Body Reconstruction agent receives the photos, runs Real-ESRGAN upscaling, MODNet background
removal, Depth Anything V2 depth estimation, ViTPose-H pose extraction, HMR 2.0 + OSX SMPL-X
parameter regression, and IHRPN surface refinement. The resulting GLB mesh and SMPL-X parameters
are stored in PostgreSQL and Supabase Storage. A 3K-polygon LOD version is streamed to the Flutter app
via WebSocket for immediate rendering.
While the avatar is rendering, the Fashion Design agent constructs an SDXL prompt from the design brief,
body parameters, and skin-tone color analysis. ControlNet OpenPose conditioning ensures the outfit
respects body proportions. LCM-LoRA generates a 512px preview in 2 seconds (streamed immediately),
followed by a full 1024px render in 8 seconds. Four variants are generated, CLIP-scored, and filtered
(score threshold 0.28).
Parallel to final diffusion, the Product Retrieval agent segments the generated outfit with SAM 2, encodes
each garment with FashionCLIP, and searches Qdrant with hybrid retrieval (dense ANN + BM25) and
cross-encoder reranking. Products above 0.72 similarity are returned with affiliate deep links. Products
below threshold trigger the Tailoring Guide agent, which generates fabric recommendations, yardage from
SMPL-X measurements, and construction instructions via the finetuned tailoring LoRA.
Finally, the Response Synthesis agent compiles all results into a warm Telugu response ("Meeru kavalasina
outfit ready aindi!"), Kokoro-82M synthesizes the audio locally, and the complete response (text + audio +
outfit images + product cards + tailoring PDF) is streamed to the Flutter app. The user's thumbs-up/down
on each outfit updates the style preference vector in PostgreSQL for future personalization.

Complete Data Strategy
Dataset Collection Pipeline
The finetuning pipeline requires four distinct datasets, each serving a different model in the system.
Dataset A: Fashion VLM Instruction Dataset (for Qwen2.5-VL-7B). Source: DeepFashion2 (801K images
with category/attribute/landmark annotations), FashionGen (293K product images with descriptions),
MPV (35K multi-pose fashion images), Farfetch (800K luxury product images), and a custom Indian ethnic
wear collection of 2,000 images (sarees, lehengas, kurtas, sherwanis, dhotis) scraped from Myntra, AJIO,
and Pinterest using the provided scraper script. Target size: 150,000 instruction-response pairs.
Dataset B: Indic Conversational Fashion Dataset (for Llama 3.3 70B). Source: 10,000 fashion
conversations in Telugu, Hindi, and English code-mixed variants. Generation method: Use the base LLM
(Llama 3.3 70B via Groq) with a carefully crafted system prompt to generate 5,000 diverse fashion
conversations covering occasions (wedding, festival, office, casual, party), body types, cultural contexts,
and budget ranges. Then manually review and correct 500 examples for quality anchor points. Use the 500
reviewed examples to few-shot prompt the generation of the remaining 4,500 conversations, iteratively
improving quality. Target size: 10,000 conversations with an average of 8 turns each.
Dataset C: Indian Ethnic Wear Diffusion Dataset (for SDXL LoRA). Source: 2,000 high-resolution images
of Indian ethnic wear organized by category: sarees (500), lehengas (400), salwar kameez (300), kurtas
(300), sherwanis (200), Indo-western (300). Each image paired with a detailed caption following the
format: "A [woman/man] wearing a [color] [fabric] [garment type] with [embroidery/print/detail],
[occasion-appropriate], [body type flattering]." Captions generated using Qwen2.5-VL vision-language
model with manual review. Target size: 2,000 image-caption pairs.
Dataset D: Tailoring Instruction Dataset (for tailoring LoRA). Source: 5,000 structured tailoring
instructions covering 40 fabric types, 25 garment categories, 12 body shape adjustments, and 8 occasion
types. Generation method: Extract instructions from open-access pattern books (Burda, Ottobre), fashion
design textbooks, and YouTube tailoring tutorial transcripts. Structure each entry as: fabric_type,
garment_type, body_adjustments, yardage_formula, construction_steps, iron_settings, finishing_details.
Target size: 5,000 instruction entries.

Synthetic Data Generation
Synthetic data addresses the scarcity of labeled Indian fashion data. Four techniques are employed.
Technique 1: LLM-Generated Conversations ù Use Llama 3.3 70B to generate diverse fashion dialogues.
The prompt template includes: system persona (expert Telugu fashion stylist), user persona (25-year-old
woman from Hyderabad attending a wedding), conversation constraints (8 turns, code-mixed Telugu-
English, specific budget 5000 INR), and quality requirements (natural slang, cultural accuracy, practical
advice). Generate 500 conversations per day via Groq free tier. Technique 2: Diffusion-Augmented Images
ù Use SDXL inpainting to modify existing fashion images: change colors, swap prints, alter silhouettes
while preserving garment structure. This 5x augments the ethnic wear dataset to 10,000 effective training
samples. Technique 3: Body Shape Variation ù Use SMPL-X parameter perturbation to synthetically
generate diverse body types (pear, apple, hourglass, rectangle, inverted triangle) wearing the same outfit,
creating body-type-specific training pairs. Technique 4: Cross-Lingual Backtranslation ù Translate English
fashion conversations to Telugu/Hindi using NLLB-200, then backtranslate to English, keeping pairs
where semantic meaning is preserved. This generates 3,000 additional Telugu fashion conversations.
Data Cleaning Pipeline
The cleaning pipeline runs as a sequential processing DAG with the following steps. Step 1: Deduplication
ù Compute MinHash LSH fingerprints for all text entries. Remove exact duplicates and near-duplicates
(Jaccard similarity greater than 0.85). For images, compute perceptual hashes (pHash) and remove
duplicates within Hamming distance 8. Step 2: Quality Filtering ù Text: Remove entries with fewer than 20
tokens, more than 2,000 tokens, or containing more than 30% non-alphanumeric characters. Compute
perplexity using a small language model (Gemma 2B) and remove entries with perplexity above the 95th
percentile (indicating low-quality or nonsensical content). Images: Remove images with resolution below
512x512, aspect ratio outside 0.5-2.0, or blur score (Laplacian variance) below 100. Step 3: Safety Filtering
ù Run images through a NSFW classifier (based on open-source NudeNet) and remove any flagged

        # Step 1: Deduplicate

        # Size filter, aspect ratio filter, blur filter, NSFW filter

                continue
            cleaned.append((img_path, caption))

Augmentation Methods
Four augmentation strategies increase effective dataset size by 4-8x without additional collection cost.
Text augmentation: back-translation (English to Telugu/Hindi and back), paraphrasing using Llama 3.3,
synonym replacement for non-technical fashion terms, and conversation truncation (keeping first N turns
to create shorter variants). Image augmentation: horizontal flip, random crop (maintaining garment
center), color jitter (hue +/- 10%, saturation +/- 15%, brightness +/- 10%), and SDXL-based style transfer
(converting photorealistic images to sketch/watercolor variants for style robustness). Instruction
augmentation: rephrase the same instruction 3 different ways (direct, polite, casual) while keeping the
response identical. Code-mixing augmentation: programmatically insert English words into Telugu/Hindi
text at natural code-switching points using a bilingual dictionary.
Instruction Tuning Format
All text datasets are converted to the ShareGPT conversation format supported by both Unsloth and
LLaMA-Factory. Each entry is a JSON object with a conversations array containing turn objects with from
(human/gpt) and value fields. For the fashion VLM, the format extends to include image URLs in the value
field using markdown image syntax. For the tailoring dataset, the format uses a structured JSON output
schema embedded in the assistant response.
Evaluation Dataset Creation
A held-out evaluation dataset of 1,000 examples (500 for each language) is created from the final cleaned
dataset using stratified sampling across all categories (occasion, body type, budget range, garment type).
This evaluation set is never seen during training and serves as the primary quality gate for model
deployment. Additionally, a regression test suite of 50 manually crafted edge cases (unusual body types,
rare occasions, ambiguous instructions) is maintained separately and must pass at 100% before any model
deployment.

DPO/RLHF for Preference Alignment
After supervised fine-tuning, the conversational model undergoes Direct Preference Optimization (DPO)
to align with user preferences. DPO is selected over PPO because it requires no separate reward model
(saving VRAM and training time) and achieves comparable alignment quality. The preference dataset
consists of 5,000 pairs: each pair contains a fashion query, a preferred response (high-quality, culturally
appropriate), and a rejected response (lower quality or generic). Preferred responses are generated by the
SFT model with temperature 0.3 (more focused). Rejected responses are generated with temperature 1.2
(more random) or are responses from the base model before fine-tuning. DPO training uses beta=0.1 (KL
penalty), learning rate 5e-5, single epoch.
Continual Learning
The system implements weekly continual learning using the ReLoRA method: adapter weights from the
previous week are merged into the base model at 10% strength, and a new LoRA adapter is trained on the
past week's user interactions (thumbs-up conversations only). This approach prevents catastrophic
forgetting while adapting to evolving user preferences and seasonal fashion trends. The merge formula is:
merged_base = base_model + 0.1 * previous_adapter. The new adapter is trained for 500 steps on the
week's positive interactions.

Adapter Stacking
Multiple LoRA adapters are composed at inference time using additive merging. For the 8B conversational
model, three adapters are stacked: the base Indic chat adapter (weight 1.0), the tailoring knowledge
adapter (weight 0.3, activated only for tailoring queries), and the user preference adapter (weight 0.5,
personalized per user). The merge is performed in VRAM before inference: merged_weights = base + 1.0 *
chat_adapter + 0.3 * tailoring_adapter + 0.5 * user_adapter. This allows a single model to serve all
conversational tasks without loading separate models.

Gradient Checkpointing and Mixed Precision
Gradient checkpointing is enabled for all training runs, trading compute for memory by recomputing
activations during backward pass rather than storing them. This reduces VRAM usage by approximately
30% at the cost of 20% slower training. Mixed precision training uses bf16 (bfloat16) on Ampere GPUs
(A100, L4) and fp16 on older GPUs (T4, P100). The GradScaler from PyTorch AMP handles loss scaling
automatically to prevent gradient underflow.

Knowledge Distillation
For the fastest query path, a student model (Qwen2.5-1.5B or Gemma 3 4B) is trained via knowledge
distillation from the teacher (Qwen2.5-VL-7B or Llama 3.3 70B). The student is trained on the soft targets
(logits) from the teacher rather than hard labels, preserving the teacher's uncertainty and fine-grained
predictions. Distillation uses temperature=2.0, alpha=0.5 (balance between soft and hard loss), and trains
for 3 epochs. The resulting student model achieves 85% of teacher quality at 10x inference speed, handling
80% of simple queries (greetings, FAQ, basic styling advice) while the teacher handles complex multi-step
reasoning.

Automated Evaluation with DeepEval
